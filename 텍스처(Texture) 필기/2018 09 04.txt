< 서브리소스 >

- 텍스처들은 배열처럼 쓸 수 있다.
밉맵 텍스처들의 각각을 서브리소스라고 부르다.
=> 배열의 인덱스는 0부터 밉맵 레벨까지 순서대로, 그 다음
텍스처 서브리소스부터 다시 시작

- D3D12CalcSubresource : 서브리소스의 배열 인덱스를 리턴해줌.

UINT PlaneSlice : Depth-Stencil 버퍼에 사용

1. File (DDS)을 읽어 들인다.
2. 텍스처를 만든다. 
3. View로 만든다. => 포맷을 바꾸지는 않는다.
만약 포맷을 바꾸려면, 파일을 읽기 전에 바꾼다. => DXGI 포맷
랜더타겟과 같게함

뷰를 만들 때, 밉맵에 관한 정보를 넣을 수 있다.

- D3D12_DEFAULT_SHADER_4_COMPONENT_MAPPING : 텍스쳐에 있는 
그대로 RGBA 순서로 매핑하는 매크로
=> 디폴트를 안쓰면 컴퍼넌트 번호를 따로 지정할 수 있다.

- 루트시그니쳐 : 루트 파라메터의 배열

루트파리메터 타입이 SRV이고, 디스크립터 테이블을 통해서만 View를
넘겨줄 수 있다. 

- D3D12_ROOT_DESCRIPTOR : 포인터를 통해 데이터를 다른 함수에 넘기는 
방식과 비슷하다. 따라가면 리소스가 나옴 
=> 텍스쳐는 DESCRIPTOR 테이블로만 넘길 수 있다.
- D3D12_ROOT_CONSTANTS : Call By Value 방식으로 밖에 동작하지 않음.

파라메터의 테이블은 이름이 디스크립터 테이블이다.(디스크립터 배열)
=> 디스크립터 테이블은 디스크립터 힙에 존재함. 따라서 디스크립터 테이블은
디스크립터의 주소이다.

- DESCRIPTOR_RANGE : 디스크립터들의 배열이 많아서 구분하기 위해 사용
